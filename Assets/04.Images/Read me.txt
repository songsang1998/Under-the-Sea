1단계 작업 가이드 - 확인 부탁.

Canvas (UI) 탭에는 인벤토리/화살표/게임오버/이미지만 일단 얹어놓음. 각각의 추가사항.
- Inventory : 상자 6개 얹어놓았음. 클릭 후 활성화이므로 각각의 상자는 "버튼"임.
- Inven_active : 좌측의 인벤토리 활성화 버튼. 이걸 누르면 인벤토리가 껐다 켜졌다를 할 수 있어야 함. 단, 지금은 꺼진 거 켜지는 것밖에 안되는 상황(SetActive)
- speaking : 대화창. 터치시 다음 대사 이동 + 대사 끝나면 닫기 구현이 필요. 모든 대사는 써놓지 않았으니 참고..
- move : 카메라 이동을 위한 화살표 버튼. 수치 조정했을때 10.5 이동하면 딱 1구간씩 이동함. 이동버튼 구현 부탁합니다!
- Game over : '돌아가기' 버튼 시 해당 층 다시 시작 / '과수원으로' 버튼 시 지난번 말했던 죽음 유형 모아놓는 복숭아밭으로 이동. 타이틀 씬으로 이동해도 괜찮은듯.
- Image : 오브젝트 조사 후 등장하는 이미지들 모음. 이름 형식은 통일해두었으니 각각 오브젝트 이름을 대응해서 찾음 됨 (Image_이름_번호 = Object_이름_번호)
 	- 각 이미지에는 임시로 닫기 버튼을 달아두었음. 삭제가능.
	- Image_lock에는 번호를 움직일 수 있는 버튼과 (현재는 움직일 수 없지만) 숫자 텍스트 0~9를 3개 박아놨음. 
	  다만 이미지 주제에 상호작용이 가능한 이 놈을 어디로 분류할 지 모호하긴 함.

Background : 각 구간 배경.
- Floor_1 : 맨 아래
- Floor_2 : 중간
- Floor_3 : 맨 위.

Item : 아이템...인데 걍 오브젝트 스프라이트로 해놓음. 크기는 적당히 맞춰서 쓰십쇼 작업 제대로 들어갈 때는 전부 통일해서 드릴거임 ㅇㅇ. (약 100*100 일케)

GameObject : 게임 내 상호작용 가능한 오브젝트들.
- 상호작용 구간 기준으로 정리해놓음 (Background 참조)
- 각 오브젝트들의 구조는 모두 통일되었으며, 다음과 같음.
폴더(오브젝트이름) - normal / active( active_1, active_2...) - (+ Child로 안에 들어 있는 오브젝트 넣어둠)
즉 Objcet_이름은 폴더이며, 찾기용으로 분류해놓은 것임.
normal은 상호작용 안한 상태. 벽장을 예시로 하면, 닫힌 벽장.
active는 상호작용 한 상태. 벽장을 예시로 하면, 열린 벽장.
active_1/2/3...이런 형태는 상호작용이 중첩된 상태. 창문을 예시로 하면, 일반창문(노멀) - 반쯤 부서진 창문(액티브1) - 다 부서져서 밖이 보이는 창문(액티브2)
대개의 오브젝트들은 normal과 active로 구성되었으나, 종종 normal만 있는 경우가 있다. 이는 상호작용을 해도 모양이 변하지 않는 오브젝트인 것임.

Child의 오브젝트들 역시 오브젝트 구조로 통일해두었으니 참조.
